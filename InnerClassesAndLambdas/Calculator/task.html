Calculator: Full On lambdas

In this task you will implement a calculator using a lot of lambda expressions.

First, finish coding the DoMath functional interface. It should have one method:compute that takes two Double objects and returns a Double object. (not a double)

Then, the Claculator object is divided in two methods:

  • toOperation will basically look at the operation sign and return the correct function to apply to any two numbers. For example, if the sign is "+" it should return a function that computes x+y. Do this for all the cases in the switch statement.
    Implementing the correct function is basically coding a short-hand implementation of DoMath's compute method
  • For the division case, if the second number is zero, return Double.POSITIVE_INFINITY. Otherwise return x/y.
  • For the default case (i.e. some operation that we have not coded), return a function that simply returns Double.MIN_VALUE
  • The compute method should set the calc value to the correct function depending on op. (that is, set calc to the correct implementation of DoMath
  • The compute method should then return the result of calling compute from the calc object with the two numbers provided in the parameters.
  • In the Task object you must create a static inner class Triplet. This class must have three generic variables that are public. The constructor of Triplet should initialize these variables.
  • Also, in the Task object, in the main method, add a few operations (division and multiplication) just to check they work ok.
  • Lastly, in the Task's main, where the forEach method is called, implement a lambda to append the result of the compute method of the Calculator c object with the elements of a Triplet as parameters, to the member variable result.
